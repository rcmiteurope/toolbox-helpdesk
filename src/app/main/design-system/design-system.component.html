Use Case 1: Manual Global Loader

Manually trigger the global loading state for long-running non-HTTP tasks (e.g., complex calculations, local initialization).

Use Case 2: Parallel Requests (Counter)

The LoadingService uses a counter. If multiple tasks start, the loader stays active until every task finishes.

Use Case 3: Sequential Intermittent Loading

Shows how the UI reacts when loading states toggle on and off in sequence.

Use Case 5: Component-Specific Loading

Individual components can track their own loading states without affecting each other. The global isLoading will still be true if any component is loading.
